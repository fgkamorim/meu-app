⬆ Importar / Restaurar Backup

⚠ Atenção: A importação sobrescreve os arquivos atuais do sistema. Use apenas backups gerados por este sistema. Faça um backup antes de importar.
Apenas arquivos .zip gerados por este sistema são aceitos.

O que é restaurado:

  • ✅ app.py — lógica do sistema
  • ✅ templates/ — páginas HTML
  • ✅ static/ — estilos e scripts
  • ✅ database.db — banco de dados
  • ❌ Arquivos fora dessas pastas são ignorados
  • ❌ Arquivos com extensões não permitidas são bloqueados

ℹ O que entra no backup completo?

  • 📄 app.py — toda a lógica e rotas do sistema
  • 🗂 templates/ — todas as páginas HTML
  • 🎨 static/ — CSS e arquivos de estilo
  • 🗃 database.db — todos os seus dados financeiros
  • 📋 pyproject.toml, replit.md e arquivos de configuração

🔒 Segurança da importação

  • ✅ Apenas arquivos .zip são aceitos
  • ✅ Path traversal é bloqueado automaticamente
  • ✅ Somente extensões seguras são restauradas (.py, .html, .css, .js...)
  • ✅ Pastas fora da lista permitida são ignoradas
  • ⚠ Sempre faça um backup antes de restaurar

💡 Boas práticas

  • 📅 Faça backup regularmente, especialmente antes de alterações
  • ☁ Guarde os arquivos em local seguro (nuvem, HD externo)
  • 🔄 Teste a restauração periodicamente para garantir integridade
  • 🗃 O banco de dados contém todos os seus dados — priorize ele
